# Propuesta Institucional

## Sistema de Inventario Digital de Plantas Medicinales de Puerto Rico

---

## 1. Resumen ejecutivo

Se propone el desarrollo de un sistema digital para la documentación, organización y divulgación de plantas medicinales de Puerto Rico mediante un modelo de **co-creación académica supervisada**.

El sistema permitirá que estudiantes generen fichas documentadas y contextualizadas que, tras un proceso de validación curatorial, pasarán a formar parte de un repositorio digital del museo.

Este proyecto integra:

* docencia
* investigación
* curaduría
* divulgación científica y cultural

---

## 2. Justificación

Puerto Rico posee un amplio acervo de conocimiento relacionado con el uso de plantas medicinales, gran parte del cual:

* se transmite oralmente
* no está sistematizado
* corre riesgo de perderse

Simultáneamente, los estudiantes universitarios requieren experiencias de aprendizaje que:

* integren investigación real
* promuevan pensamiento crítico
* produzcan resultados con impacto

Esta propuesta articula ambas necesidades mediante la creación de un sistema sostenible de producción de conocimiento.

---

## 3. Objetivos

### Objetivo general

Desarrollar un inventario digital estructurado de plantas medicinales de Puerto Rico mediante un modelo de participación estudiantil supervisada.

---

### Objetivos específicos

* Diseñar una estructura de ficha estandarizada
* Implementar un sistema de entrada de datos accesible
* Establecer un flujo de validación curatorial
* Crear una interfaz pública de consulta
* Generar un repositorio en crecimiento continuo

---

## 4. Modelo conceptual

El sistema se basa en una estructura de tres niveles:

### Nivel 1 — Identidad

* nombre común
* nombre científico
* imagen
* resumen museográfico

### Nivel 2 — Uso

* usos tradicionales
* preparación
* advertencias

### Nivel 3 — Profundización

* dimensión científica
* medios adicionales
* referencias

Este modelo permite integrar:

* conocimiento tradicional
* análisis científico
* narrativa cultural

---

## 5. Modelo pedagógico

El proyecto se fundamenta en un esquema de:

> Docente – Estudiante – Sistema digital

Donde:

* el estudiante investiga y produce
* el sistema organiza y estructura
* el docente y el curador validan

La ficha final funciona como:

* evidencia de aprendizaje
* producto de investigación
* recurso educativo público

---

## 6. Flujo de trabajo

### Producción

```text
Estudiante investiga
↓
Completa ficha
↓
Valida mediante vista previa
```

### Validación

```text
Curador revisa
↓
Aprueba / solicita cambios / rechaza
```

### Publicación

```text
Ficha aprobada
↓
Se integra al repositorio del museo
```

---

## 7. Roles institucionales

### Estudiantes

* generan contenido
* realizan investigación
* producen fichas

### Profesores

* guían el proceso
* evalúan el trabajo
* integran el proyecto al curso

### Curadores / Museo

* validan contenido
* mantienen calidad
* gestionan publicación

---

## 8. Plataforma tecnológica

Se propone el uso de:

* WordPress como sistema base
* Advanced Custom Fields para estructuración de datos
* H5P para contenido interactivo

El sistema incluirá:

* formularios de entrada
* control de roles
* estados editoriales
* gestión de medios

---

## 9. Valor institucional

Este proyecto permite:

* integrar cursos con producción real
* fortalecer el vínculo universidad–museo
* crear un archivo digital de valor cultural
* fomentar investigación interdisciplinaria
* visibilizar conocimiento tradicional

---

## 10. Sostenibilidad

El sistema está diseñado para:

* crecer con cada cohorte de estudiantes
* requerir mínima intervención técnica
* distribuir la carga de producción
* mantener control editorial centralizado

---

## 11. Impacto esperado

* creación de un inventario digital robusto
* fortalecimiento de competencias investigativas
* preservación de conocimiento cultural
* generación de recursos educativos abiertos

---

## 12. Fases de implementación

### Fase 1 — Prototipo (completada)

* modelo de datos
* interfaz inicial
* flujo conceptual

### Fase 2 — Implementación técnica

* desarrollo en WordPress
* creación de formularios
* configuración de roles

### Fase 3 — Piloto académico

* integración en cursos
* primeras cohortes de estudiantes

### Fase 4 — Expansión

* ampliación del repositorio
* integración con otras disciplinas

---

## 13. Conclusión

Esta propuesta transforma el proceso educativo en un sistema de producción de conocimiento donde:

* el estudiante investiga
* el docente guía
* el curador valida
* el museo preserva

El resultado es un repositorio vivo que articula ciencia, cultura y educación en un mismo espacio digital.
